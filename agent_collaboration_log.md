## Section 4: Ethical Reflection

### What the AI did well:
- **Code scaffolding**: Generated the Elo rating loop, Bayesian shrinkage pipeline, Optuna tuning infrastructure, SHAP computation, portfolio backtest loop, and all matplotlib/seaborn visualisation code. These are mechanically complex but intellectually straightforward - exactly the tasks where AI assistance saves the most time without compromising originality.
- **Debugging support**: When SHAP TreeExplainer failed on RF arrays, the agent generated four alternative configurations before the interventional approach worked. When yfinance returned inconsistent column formats, the agent produced a format-detection wrapper. The iterative debugging cycle was productive.
- **Documentation and formatting**: The agent generated the LaTeX equation formatting (Kelly, ECE, Elo update), the python-docx figure embedding, the report table formatting, and the README structure. These are presentation tasks where the agent's output was accepted with minimal modification.

### What the AI could not do:
- **Frame the research question**: The decision to study calibration rather than accuracy, and the choice of Kelly sizing as the economic motivation, were entirely human-directed. The agent would have built an accuracy-optimised pipeline without questioning whether accuracy was the right metric.
- **Enforce temporal discipline**: The agent's first instinct was `KFold(shuffle=True)`. Every temporal safeguard - the calendar-order split, the expanding-window CV, the leakage assertion, the one-touch test set - was human-initiated. The agent does not understand that financial data has a time arrow.
- **Make domain judgements**: Feature selection (why Elo captures persistence), metric interpretation (why ECE matters more than AUC for Kelly sizing), and result framing (why the null result is honest, not a failure) all required financial reasoning the agent could not replicate.
- **Assess statistical rigour**: The agent compared Sharpe ratios without testing significance, produced evaluation figures on training data, and omitted power analysis. Every piece of statistical discipline was added by the student.
- **Recognise its own failures**: The agent printed "All VIFs < 5.0" after displaying VIFs of 2,809. It generated training-set ROC curves labelled as "model evaluation." It never flagged the missing test-set figures. The pattern is consistent: the agent produces outputs that look correct but require human verification to confirm they are correct.

### Honest assessment:
The agent accelerated the technical implementation by an estimated 15–20 hours over three weeks. However, the intellectual contributions - hypothesis formulation, temporal design, metric selection, statistical testing, domain interpretation, and all business-facing narrative - were human-directed throughout. The most instructive moments were the agent's failures: the temporal leakage attempt (D4) taught that ML defaults are not finance-appropriate defaults, the missing significance test (D8) demonstrated that the agent treats every numerical difference as a finding, and the 46% ECE degradation (only visible because I explicitly requested test-set figures) became the project's most important result. The agent was a powerful tool that amplified productivity, but every output required verification, and the most consequential decisions remained firmly with the student.
